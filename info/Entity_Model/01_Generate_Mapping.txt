Создаем Entity, кто-то их еще называет моделями, кто-то доменами.

В Idea есть возможность несколько автоматизировать,
сгенерировать создание сущностей (моделей) из БД в том случае,
если у вас уже создана база данных и подключена к Idea.

Для этого открываем вкладку
View > Tool Windows > Persistence
кликаем правой кнопкой на строку с названием нашей базы
(см. persistence_tab.png)
и выбираем: Generate Persistence Mapping > By Database Schema

----------------------------------

Окно Import Database Schema

В разделе General Settings
В меню Choose Data Source выбираем наше подключение к базе данных

В меню Package выбираем или создаем папку где будут лежать
наши модели (сущности), в моем случае это: src/main/java/task/entity

По желанию делаем префикс или суффикс у названий классов
(Entity prefix, Entity suffix)

Снимаем галочку: Prefer primitive types
(Нам нужны объекты, а не примитивы, чтобы можно
было записать в эти поля NULL)

Ставим галочку: Show default relationships
Выставляются все связи между таблицами (сущностями),
(потом лишние связи можно бужет убрать).

В меню Database Schema Mapping нажимаем на галочку: (Select All)
или вручную отмечаем все таблицы, из которых надо сделать сущности

В разделе Generation Settings
Ставим галочки на:
Generate JPA Annotations (Java 5)
Generate Column Properties

(см. import_database_schema.png)

Далее нажимаем OK

--------------------------------

Примечание 1. Если вам не понравился результат,
классы можно удалить и попробовать сгенерировать заново.

После генерации сущностей (моделей) их нужно отредактировать.
Я использую аннотации Lombok для уменьшения шаблонного
(boilerplate) кода

Примечание 2. Если названия колонок в аннотации
@Column подчеркиваются красным, значит возможно
нет связи с базой данных

Примечание 3. Для улучшения связи между сущностями
и таблицами нужно во вкладке Persistence правой кнопкой
кликнуть опять на нашу связь и выбрать: Assign Data Source,
в появившемся окне выбрать связь и нажать ОК.
После этого, кликая, например, вместе CTRL на id в моделях
@Column(name = "id")
Idea показывает вам соответствующую связанную колонку
в соответствующей таблице на вкладке Database.

Примечание 4. Мне кажется применение этого метода
для создания моделей очень полезно для обучения
или проверки знаний по работе с моделями.
