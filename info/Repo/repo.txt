Примечания к репозиториям

Создаем репозитории в папке repo

При нажатии CTRL+ЛКМ  (левая кнопка мыши) на JpaRepository
мы можем увидть какие интерфейсы он расширяет
PagingAndSortingRepository и QueryByExampleExecutor
Если мы на интерфейсе нажмем ПКМ (правую кнопку мыши),
и выберем: Go To > Implementation
мы можем посмотреть наоборот, какие классы или интерфейсы
используют данный интерфейс

PagingAndSortingRepository позволяет выводить
данные с постраничным выводом, и есть возможность сортировки
Этот интерфес расширяет CrudRepository, которые предоставляет
базовые методы CRUD

QueryByExampleExecutor - позволяет более удобно работать
с запросами

В репозитории src/main/java/task/repo/CategoryRepository.java
мы реализуем для примера работу с JPQL
в методе
List<Category> findByTitle(@Param("title") String title);
Помимо стандартных возможностей CRUD предоставляемых интерфейсом
можно добавить запросы с помощью
JPQL (Java Persistence Query Language), который похож на SQL,
но с некоторыми особенностями.
Про параметры.
Когда поступает String title, @Param передает его в параметр "title",
и этот параметр будет использоваться в SQL запросе.
SQL запрос.
Для создания SQL запроса используется @Query. Этот SQL
запрос будет вызываться, когда будет вызываться метод.

JPA Named Queries
Можем создавать методы с нужными названиями, по которым
Spring/Hibernate будет создавать SQL запросы.
В запросе указываются поля и метод сортировки.
Для примера заменим метод findAll в классе
src/main/java/task/repo/PriorityRepository.java
